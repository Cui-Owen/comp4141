### Q1 — FSA：画 NFA + Subset Construction

**画 ε-NFA 的套路（Thompson 四条）**

| 结构 | 怎么画 |
|------|--------|
| 单符号 a | q₀ →a→ q₁（q₁ 接受） |
| 连接 R₁R₂ | R₁ 接受态 →ε→ R₂ 起始态 |
| 并 R₁∪R₂ | 新起始分两条 ε 到 R₁、R₂；两个接受态汇合到新接受 |
| R* | 新起始 →ε→ 子NFA起始；子NFA接受 →ε→ 子NFA起始（回环）；新起始 →ε→ 新接受（跳过，接受空串） |

**考场推荐：别死套 Thompson，直接按语言结构画**

拿 1(0∪01)∗1 举例，5 个状态 q₀…q₄：
- q₀ →1→ q₁（开头的 1）
- q₁ →0→ q₂（进循环）
- q₂ →ε→ q₁（走"0"这条路回去）
- q₂ →1→ q₃ →ε→ q₁（走"01"这条路回去）
- q₁ →1→ q₄（结尾的 1，接受态）

验证路径必须写："11" q₀→q₁→q₄ ✓ / "101" q₀→q₁→q₂→ε→q₁→q₄ ✓ / "1011" q₀→q₁→q₂→q₃→ε→q₁→q₄ ✓

**Subset Construction 怎么做**

1. 先把每个状态的 ECLOSE 算好（q 自己 + 顺着 ε 边能到的所有状态）
2. DFA 起始 = ECLOSE(NFA 起始)
3. 每一格 = 先走符号边到的状态集合，再对这个集合取 ECLOSE
4. DFA 接受态 = 只要子集里**含有**任何一个 NFA 接受态就算

**易错**：ECLOSE 漏了中间状态 / ε 边方向搞反 / 接受态不是只有 {q₄}，是所有含 q₄ 的子集

### Q2 — Pumping Lemma（证不是 regular）

**定理**：L 是 regular → 存在 p ≥ 1，对任意 w ∈ L 且 |w| ≥ p，都能拆成 w = xyz 满足：|xy| ≤ p，|y| ≥ 1，对所有 i ≥ 0 都有 xyⁱz ∈ L

**四步写法（直接套）**：
1. 假设 L 是 regular，设 p 是 pumping length。
2. 选 w = [你的串]（要保证 |w| ≥ p 且 w ∈ L）
3. 由 Pumping Lemma，w = xyz，|xy| ≤ p，|y| = s ≥ 1。
4. 取 i = [通常 2]：推导出 xyⁱz ∉ L，矛盾。所以 L 不是 regular。□

**各种语言怎么选词、怎么卡不等式**

| 语言 | 选词 | pump 后 | 夹逼不等式 |
|------|------|---------|-----------|
| {a^{2ⁿ}} | a^{2ᵖ} | a^{2ᵖ⁺ˢ} | 2ᵖ < 2ᵖ+s ≤ 2ᵖ+p < 2·2ᵖ = 2^{p+1}，卡在两个 2 的幂之间 |
| {aⁿ²} | a^{p²} | a^{p²⁺ˢ} | p² < p²+s ≤ p²+p < (p+1)²，卡在两个完全平方之间 |
| {aⁿ!} | a^{p!} | a^{p!⁺ˢ} | p! < p!+s ≤ p!+p < (p+1)!，卡在两个阶乘之间 |
| {aⁿbⁿ} | aᵖbᵖ | pump up/down | |xy| ≤ p 说明 y 全在 a 那段，pump 之后 a 的数量变了但 b 没变 |
| {aⁿbⁿcⁿ} | aᵖbᵖcᵖ | 同上 | y 只落在某一段里，三段数量对不上 |

**关键**：s ≤ p 是因为 |xy| ≤ p 而 y 在 xy 里面，不是因为 |w|。

pump down（i=0）：适合"长度不能跳过某个值"的情况，比如素数长度的语言。

**Myhill-Nerode（另一种证不是 regular 的方法）**

两个串 x、y 对 L **可区分**：存在 z，使得 xz ∈ L 但 yz ∉ L（或反之）。
若能找到无穷多个两两可区分的串 → L 不是 regular。

套路：构造串族 S = {aⁿ | n ≥ 0}，对任意 i ≠ j，取区分串 z = bⁱ：
- aⁱbⁱ ∈ L（若 L = {aⁿbⁿ}）但 aʲbⁱ ∉ L → aⁱ 与 aʲ 可区分。
因为 S 无穷大，L 有无穷多等价类 → 不是 regular。□

写法要点：说清楚（1）区分串 z 是什么，（2）xz 在 L 里、yz 不在 L 里，（3）因此任意两个串不等价，（4）所以等价类无穷多。

### ▌CFL Pumping Lemma（证不是 context-free）

**定理**：L 是 CFL → 存在 p，对任意 w ∈ L 且 |w| ≥ p，w = uvxyz 满足：
|vxy| ≤ p，|vy| ≥ 1，对所有 i ≥ 0 有 uvⁱxyⁱz ∈ L

**与 regular PL 的区别**：拆成五段，v 和 y **同步** pump（都变 i 份）；x 是中间固定段。

**四步写法**（同 regular PL，但第 3 步改成五段）：
1. 假设 L 是 CFL，设 p 是 pumping length。
2. 选 w = [具体串]，满足 |w| ≥ p 且 w ∈ L。
3. 由 CFL PL，w = uvxyz，|vxy| ≤ p，|vy| ≥ 1。
4. 取 i = 0 或 2，推导 uvⁱxyⁱz ∉ L，矛盾。□

**常考语言和反驳方式**

| 语言 | 选词 | 为什么矛盾 |
|------|------|-----------|
| {aⁿbⁿcⁿ} | aᵖbᵖcᵖ | |vxy| ≤ p，v 和 y 最多跨两段；pump 后第三段数量不变，三段失衡 |
| {aⁿbⁿcⁿdⁿ} | aᵖbᵖcᵖdᵖ | 同理，四段更不可能同步 |
| {0^{2ⁿ}} | 0^{2ᵖ} | pump 后长度夹在两个 2 的幂之间，不在语言里 |
| {ww \| w∈{0,1}*} | 0ᵖ1ᵖ0ᵖ1ᵖ | v 和 y 被限制在前半或后半内，pump 后前后不再相同 |

**关键**：pump 后用数量不等、或结构不对称来导出矛盾；|vxy| ≤ p 限制了 v 和 y 只能落在某个长度 ≤ p 的窗口内。

### Q3 — CFG 设计 + 证明

**常用文法套路**

| 条件 | 怎么写 |
|------|--------|
| n = m | S → aSb \| ε |
| n > m | S → aSb \| aA；A → aA \| ε |
| n < m | S → aSb \| bB；B → bB \| ε |
| n − m ≥ 2 | S → aSb \| aaA；A → aA \| ε（出循环时多吐两个 a） |
| 两段独立（aⁿbᵐcᵐ） | S → aS \| T；T → bTc \| ε |
| 两段独立（aⁿbⁿcᵐ） | S → XY；X → aXb \| ε；Y → cY \| ε |

**正确性证明必须写两个方向！**

**L(G) ⊆ L**（文法生成的串都合法）：追踪推导过程，数清楚参数。
> Example: "A derivation applies S→aSb k times, then S→aaA, then A→aA j times (j ≥ 0), yielding a^{k+2+j}b^k. Here n−m = 2+j ≥ 2 ✓"

**L ⊆ L(G)**（合法的串都能被生成）：给定 w = aⁿbᵐ ∈ L，反推怎么用文法生成它。
> Example: "Let k = m, j = n−m−2 (j ≥ 0 since n ≥ m+2). Then w = a^{k+2+j}b^k is generated by S ⇒ᵏ aᵏS₁bᵏ ⇒ a^{k+2}Ab^k ⇒ʲ a^{k+2+j}b^k. ✓"

**注意**：j ≥ 0 的理由必须写，不写扣分。

### ▌CNF 转换 + CYK 算法

**为什么要 CNF**：CYK 要求所有规则形如 A→BC 或 A→a，其他形式先转换。

**CNF 转换四步（按顺序做，不能乱）**：

1. **处理 ε-规则**：找所有可推出 ε 的非终结符（nullable），对每条含 nullable 的规则，补上省略掉 nullable 的版本。最后删掉 A→ε（保留 S→ε 如果原来 ε ∈ L）。
2. **处理单元规则**（A→B）：找出所有 A 通过单元规则能到的非终结符，把它们的规则直接拷到 A 下面，然后删掉单元规则。
3. **拆长规则**（A→B₁B₂B₃…）：引入新非终结符：A→B₁C₁，C₁→B₂C₂，C₂→B₃…
4. **孤立终结符**（长规则里的 a）：引入 Tₐ→a，把规则里的 a 换成 Tₐ。

**CYK 表格怎么填**（判断串 w = w₁w₂…wₙ 是否 ∈ L(G)）：

- 表格 T[i][j] = 从位置 i 到 j 的子串 wᵢ…wⱼ 能由哪些非终结符生成。
- **底行**（长度 1）：T[i][i] = {A | A→wᵢ 是规则}
- **往上填**（长度 l = 2, 3, …, n）：T[i][i+l−1] = {A | 存在分割点 k，A→BC 是规则，B∈T[i][k]，C∈T[k+1][i+l−1]}
- **结论**：S ∈ T[1][n] → w ∈ L(G)；否则不在。

### ▌PDA（下推自动机）设计套路

PDA = NFA + 一个栈。栈可以 push/pop，但只能看栈顶。

**状态和栈的分工**：
- 状态：记录"现在在处理哪个阶段"（读第一段 / 准备匹配 / 读第二段）
- 栈：记录"需要延迟匹配的东西"（读到 a 就 push，之后读 b 就 pop）

**三段式套路**（适用于 {aⁿbⁿ}、{aⁿbᵐcᵐ} 等）：

```
阶段 1（读 a，push）：读一个 a，push A 到栈
阶段 2（转换，不消耗输入）：ε 转换到阶段 3
阶段 3（读 b，pop）：读一个 b，pop A；读完后栈空则接受
```

**用栈底标记**：一开始 push 一个特殊符号 $ 进栈，之后若看到 $ 说明栈已空，可以用来判断"是否读完了第一段"。

**{aⁿbⁿ} 的 PDA**（3 个状态）：
- q₀ →(a, ε/A)→ q₀（读 a，push A）
- q₀ →(ε, ε/ε)→ q₁（非确定性切换）
- q₁ →(b, A/ε)→ q₁（读 b，pop A）
- q₁ 在栈空时接受（或加接受态 q₂ 通过 ε 转换）

**{aⁿbᵐcᵐ} 两段独立**：先 ε 跳过 a 不管，再用上面套路匹配 b 和 c。

### Q4 — Decidability：有限搜索

**核心思路**：如果要找的东西 u 长度有上界（比如 |u| ≤ |w|），那搜索空间就是有限的 → 一个个试 → 每个都用已知的 decider 判 → 一定能停机。

**标准写法**（以 L₂ = {w | ∃u, |u|≤|w|, wu∈L} 为例）：

> "Let M be a decider for L. Construct M' as follows:
> On input w, let n = |w|.
> For each string u with |u| ≤ n (enumerate ε, 0, 1, 00, 01, … up to length n):
> 　　Run M on wu.
> 　　If M accepts, M' accepts.
> If all such u are exhausted without acceptance, M' rejects."

**为什么一定停机**：u 的总数有限（最多 2^{n+1}−1 个），每次 M 一定停机 → M' 一定停机。
**正确性**：M' 接受 w ⟺ 存在 |u|≤|w| 使得 wu ∈ L ⟺ w ∈ L₂。

**变种**：
- |u| ≤ 2|w| 或 |u| ≤ |w|²：搜索空间更大但仍然有限，同样做法
- 把 ∃ 换成 ∀（"对所有 u 都要满足"）：改成"有任何一个不满足就 reject，全通过才 accept"

**注意**：如果题目没给长度限制（比如 PREF），搜索空间无限，这个方法就不能用了！

### ▌Undecidability：规约证明某语言不可判定

**核心工具**：mapping reduction（≤ₘ）

A ≤ₘ B 的意思：存在一个可计算函数 f，使得 w ∈ A ⟺ f(w) ∈ B。

**用来证不可判定的逻辑**：
- 如果 A 不可判定，且 A ≤ₘ B，那么 B 也不可判定。
- 因为：若 B 可判定，则 A 也可判定（先算 f(w)，再用 B 的 decider 判 f(w)），矛盾。

**最常用的起点（可直接引用）**：
- ATM = {〈M, w〉 | M 是 TM 且 M 接受 w} — 不可判定（图灵定理）
- HALT = {〈M, w〉 | M 在 w 上停机} — 不可判定

**规约写法模板**（证语言 L 不可判定）：

"We show ATM ≤ₘ L.
Given 〈M, w〉, construct 〈M'〉 as follows: M' on input x does [具体行为，通常是先模拟 M 跑 w，然后…].
f is computable: [一句话说为什么].
Correctness: 〈M, w〉 ∈ ATM ⟺ M accepts w ⟺ [M' 的行为] ⟺ 〈M'〉 ∈ L.
Since ATM ≤ₘ L and ATM is undecidable, L is undecidable. □"

**Rice 定理（直接引用，不用证）**：
对任何非平凡的语言性质 P（即既不是所有 TM 都满足，也不是都不满足），
{〈M〉 | L(M) 满足 P} 都不可判定。

能用 Rice 的前提：P 描述的是 L(M)（语言的性质），不是 M 本身的行为（如"M 有多少状态"就不算）。

**常考变种**：

| 语言 | 规约方向 | 关键构造 |
|------|---------|---------|
| {〈M〉 \| L(M) ≠ ∅} | ATM ≤ₘ 它 | M' 忽略输入，直接模拟 M(w)；接受则接受任何输入 |
| {〈M〉 \| L(M) = Σ*} | ATM ≤ₘ 它 | 同上，接受所有输入改为只接受固定集合 |
| {〈M〉 \| M 在某串上停机} | ATM ≤ₘ HALT | 直接规约 |

**Turing-recognizable（可识别但不可判定）**：
- 可以写一个 TM 跑，接受的串都在 L 里，但对不在 L 里的串可能不停机（无法 reject）。
- {〈M〉 | L(M) ≠ ∅} 是 recognizable 但不 decidable：枚举所有输入，挨个模拟（dovetailing）。
- coRE = recognizable 的补集的语言类；decidable = RE ∩ coRE。

### Q5 — 对角化（Diagonalization）

**四步走**

**Step 1 — 枚举**：DFA/NFA/CFG 都是有限描述的对象，可以按长度+字典序排成 M₁, M₂, M₃, …
> "All DFAs can be enumerated as M₁, M₂, … (ordered by encoding length, then lexicographically among equal-length encodings). Let E be a TM that on input k outputs 〈Mₖ〉; E always halts."

**Step 2 — 定义 K**：
K = {0ᵏ | 0ᵏ ∉ L(Mₖ)}
（第 k 个全零串在不在 K 里，取决于第 k 台机器接不接受它——然后取反）

**Step 3 — 证 K 不在目标类里**（两种情况都要写！）：
> "For any k, consider 0ᵏ:
> - If Mₖ accepts 0ᵏ, then by definition 0ᵏ ∉ K, so K ≠ L(Mₖ).
> - If Mₖ does not accept 0ᵏ, then by definition 0ᵏ ∈ K, so K ≠ L(Mₖ).
>
> In both cases K ≠ L(Mₖ). Since k is arbitrary, K ≠ L(Mₖ) for every k, so K is not regular."

**Step 4 — 证 K 是 decidable**（构造 TM D）：
> "D on input w:
> 1. If w is not of the form 0ᵏ, reject.
> 2. Let k = |w|. Run E(k) to obtain 〈Mₖ〉.
> 3. Simulate Mₖ on 0ᵏ.
> 4. **Flip the answer**: if Mₖ accepts, D rejects; if Mₖ rejects, D accepts.
>
> D always halts (format check halts, E halts, DFA simulation halts).
> D accepts 0ᵏ ⟺ Mₖ rejects 0ᵏ ⟺ 0ᵏ ∉ L(Mₖ) ⟺ 0ᵏ ∈ K."

**枚举对象不同，Step 4 模拟方式不同**：

| 对象 | 怎么模拟 |
|------|---------|
| DFA | 直接跑，走 \|w\| 步就停 |
| NFA | 先转 DFA（subset construction）再跑 |
| CFG | 先转 CNF，再用 CYK 算法判定 |

**易错**：Step 3 只写了一种情况 / Step 4 忘了取反 / 干脆没写 Step 4

### Q6 — NP-Complete 证明

**两步：先证 ∈ NP，再证 NP-hard**

**Step 1 — 证 ∈ NP**（给 certificate + verifier）：
> "Certificate: [e.g. a subset C ⊆ V of k vertices].
> Verifier: On input 〈G, k〉, guess C nondeterministically, check [specific condition] in polynomial time. Therefore the problem is in NP."

**Step 2 — 证 NP-hard**（从已知 NP-hard 问题规约过来）：
> "We show [known-problem] ≤ₚ [target-problem].
> Define f(〈G, k〉) = [the transformation].
> f is polynomial-time: [justification].
> Key Lemma: [statement], proved in both directions.
> Correctness: 〈G, k〉 ∈ [known] ⟺ … ⟺ f(〈G, k〉) ∈ [target].
> Since [known] is NP-hard and [known] ≤ₚ [target], [target] is NP-hard.
> Combined with Step 1, [target] is NP-complete. □"

**三个常考规约（背下来）**

**IS → CLIQUE（取补图，k 不变）**
- f(〈G, k〉) = 〈Ḡ, k〉，Ḡ 是 G 的补图
- 引理：I 是 G 的 k-IS ⟺ I 是 Ḡ 的 k-clique
- →：I 里任意两点在 G 里没边 → 在 Ḡ 里有边 → 是 clique
- ←：I 里任意两点在 Ḡ 里有边 → 在 G 里没边 → 是 IS

**IS → VC（图不变，k 变成 |V|−k）**
- f(〈G, k〉) = 〈G, |V|−k〉
- 引理：I 是 G 的 k-IS ⟺ V∖I 是 G 的 (|V|−k)-VC
- →：I 是 IS → 每条边至少一端不在 I 里 → 至少一端在 V∖I 里 → V∖I 是 VC
- ←：V∖I 是 VC → 假设 I 里有边 → 这条边两端都不在 V∖I → 没覆盖，矛盾 → I 是 IS

**CLIQUE ↔ IS**：和 IS→CLIQUE 对称，引理一模一样

**各问题的 Certificate + Verifier 速查**

| 问题 | Certificate | 检查什么 |
|------|-------------|---------|
| IS | k 个点的子集 I | I 里任意两点之间没边 |
| CLIQUE | k 个点的子集 I | I 里任意两点之间有边 |
| VC | k 个点的子集 C | 图里每条边至少一端在 C 里 |
| 3SAT | 一组变量赋值 | 每个子句至少一个 literal 为 true |

**怎么判断用哪个规约**：
- 改图不改 k → 补图（IS ↔ CLIQUE）
- 改 k 不改图 → 集合互补（IS ↔ VC）

规约链：3SAT ≤ₚ CLIQUE ≤ₚ IS ↔ VC

**易错**：引理只写了一个方向 / 忘了说 f 是多项式时间 / 最后没把 NP + NP-hard 合起来说 NP-complete

### ▌coNP

**定义**：coNP = {L | L̄ ∈ NP}。换句话说，"反例"有多项式长度 certificate 的问题。

**代表问题**：VALIDITY = {φ | φ 对所有赋值都为真} — coNP-complete。

**证 L ∈ coNP**：证 L̄ ∈ NP，即给 L̄ 的 certificate + verifier。

**证 coNP-hard**：从 VALIDITY 规约过来（或引用其他已知 coNP-complete 问题）。

**NP 和 coNP 的关系**：
- 所有可判定语言都在 NP ∩ coNP 里（甚至更小的 P 里）
- NP = coNP 是否成立未知
- 若 NP-complete 问题 L 也是 coNP-complete，则 NP = coNP（很可能不是）

### Q7 — PSPACE

**答案一般是 PSPACE-complete，先把结论写上去**

**第一步 — 搞清状态空间有多大**

| 程序类型 | 变量范围 | 状态总数 T |
|---------|---------|-----------|
| Boolean | 每个变量 0 或 1，n 个变量 | \|P\| × 2ⁿ |
| Modular | 每个变量 0 到 n−1，k 个变量 | \|P\| × nᵏ |

鸽巢原理：跑超过 T 步一定有配置重复 → 进入死循环 → 直接 reject。

**第二步 — 证上界（∈ PSPACE）**

对固定输入 π：模拟程序跑，最多 T 步，超了就 reject。
空间只需要存当前配置（变量值 + 程序位置 + 计步器），都是多项式位数。

如果有 ∃π（"存在某个输入使程序停机"）：非确定性猜 π → 这是 NPSPACE → 用 **Savitch 定理**（NPSPACE = PSPACE）得到 ∈ PSPACE。

**Savitch 定理必须引用！**

**第三步 — 证下界（PSPACE-hard，从 TQBF 规约）**

TQBF（真的全称量化布尔公式）是 PSPACE-complete，可以直接引用。

**规约口诀：存在靠猜（读 π），任意靠跑（枚举所有值）**

| 量词 | 怎么处理 |
|------|---------|
| ∃xᵢ | 从输入 π 里读，程序不改它 |
| ∀xᵢ（Boolean） | xᵢ:=0; [子程序]; xᵢ:=1; [子程序]（代码写两遍，两个值都跑） |
| ∀xᵢ（Modular） | 用循环：xᵢ:=0; counter:=0; while counter≠n do {[子程序]; xᵢ:=(xᵢ+1) mod n; counter++} |
| 最内层公式 ψ | if ¬ψ then { while 1≠0 do {} }（假就死循环，真就正常结束） |

**例子**：φ = ∃x₁ ∀x₂ ψ(x₁,x₂)
```
// x₁ 从 π 读
x₂ := 0
[Boolean: 跑子程序; x₂:=1; 再跑子程序]
[Modular: counter:=0; while counter≠n do { 子程序; x₂++; counter++ }]
子程序 = if ¬ψ(x₁,x₂) then { while 1≠0 do {} }
```

**正确性**：∃π 使程序停机 ⟺ ∃x₁ 使所有 x₂ 下 ψ 都为真 ⟺ φ 为真。

还要说：|P_φ| 关于 |φ| 是多项式大小。

**易错**：∃ 和 ∀ 的处理搞反了 / 忘引用 Savitch / 没说构造出的程序大小是多项式 / 只写了上界没写下界
